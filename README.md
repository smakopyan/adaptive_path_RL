# Adaptive Path Planning with Reinforcement Learning

Пакет ROS2 для адаптивного планирования пути робота TurtleBot3 с использованием алгоритмов динамического программирования и оптимального управления.

## Описание проекта

Этот проект реализует систему навигации и планирования траектории для мобильного робота TurtleBot3 в окружении Gazebo. Система использует несколько алгоритмов планирования пути для обхода препятствий:

- **D* (D-Star)** - алгоритм динамического переиспользования A*, позволяющий эффективно переплани​вать маршрут при появлении новых препятствий
- **PMP (Pontryagin Maximum Principle) v2** - оптимальное управление на основе принципа максимума Понтрягина с моделью дифференциального привода

Проект интегрирован с:
- ROS2 (Robot Operating System 2)
- Gazebo (симулятор физики)
- TurtleBot3 (мобильный робот)
- OpenCV (обработка карт)

## Требования

- ROS2 (Humble или новее)
- Python 3.8+
- TurtleBot3 пакеты
- Gazebo
- OpenCV
- NumPy

## Алгоритмы планирования

### D* (D-Star)

Алгоритм улучшение A*, используется для быстрого переплани​рования при обновлении карты. Основные характеристики:
- Эффективное обновление при изменении стоимостей ребер
- Поддержка динамических препятствий
- Сложность: O(n log n) в большинстве случаев

**Класс:** `DStar` в `D_star.py`

### PMP v2 (Pontryagin Maximum Principle)

Оптимальное управление на основе принципа максимума Понтрягина. Использует модель дифференциального привода:
- Оптимальные управления (линейная и угловая скорость)
- Минимизация времени прохождения траектории
- Поддержка ограничений на управления

**Класс:** `DifferentialDriveModel`, `Hamiltonian` в `pmp_v2.py`

## Обработка карт

Система использует карты в формате:
- **PGM** (Portable Graymap) - растровое изображение карты
- **YAML** - метаинформация (разрешение, начало координат и т.д.)

Процесс обработки:
1. Загрузка карты из файлов PGM/YAML
2. Расширение препятствий с использованием морфологических операций (cv2.dilate)
3. Создание карты стоимостей для планирования

## ROS2 интеграция

### Издаваемые топики

- `/plan` - запланированная траектория
- `/costmap` - карта стоимостей
- `/markers` - визуализация в RViz

### Подписываемые топики

- `/map` - входящая карта окружения
- `/odom` - одометрия робота
- `/scan` - данные лидара

### Параметры узла

- `expansion_size` - размер расширения препятствий (пиксели)
- `map_frame` - фрейм карты
- `robot_radius` - радиус робота

## Будущие расширения

### 1. Обновление карты в реальном времени
Планируется реализация системы адаптивного обновления карты на основе данных лидара/камеры:
- Динамическое переплани​рование при обнаружении новых препятствий
- Интеграция с SLAM системами
- Поддержка полностью неизвестного окружения

### 2. Усиленное обучение (TD3 для возмущений)
Планируется добавить модуль подавления возмущений на основе алгоритма **TD3 (Twin Delayed DDPG)**:
- Обучение агента для компенсации системных ошибок и возмущений
- Улучшение робастности навигации
- Адаптация к различным условиям окружения (неровные поверхности, сбои датчиков и т.д.)
- Интеграция с генератором траектории PMP

**Примерная архитектура:**
```
Планировщик пути (D*/PMP)
    ↓
Генератор траектории (PMP)
    ↓
RL агент (TD3) - компенсация возмущений
    ↓
Контроллер робота (TurtleBot3)
```
